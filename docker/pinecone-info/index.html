# Vector Database Configuration VECTOR_DB=pinecone # Pinecone Configuration (dimension is auto-detected from embedding model) VECTOR_DB_CONFIG={"api_key": "your_pinecone_api_key", "region": "us-east-1", "cloud": "aws", "index_name": "hybrid-search", "metric": "cosine"} # Configuration Parameters: # - api_key: Your Pinecone API key from app.pinecone.io # - region: AWS region (e.g., "us-east-1", "us-west-2", "eu-west-1") # - cloud: Cloud provider ("aws", "gcp", or "azure") # - index_name: Name for your index (created automatically if doesn't exist) # - metric: Distance metric ("cosine", "euclidean", or "dotproduct") # - dimension: NOT needed - auto-detected from your embedding model

üöÄ Quick Start

  1. Sign up at pinecone.io (free starter plan available)
  2. Get your API key from the Pinecone Console
  3. Note your region (e.g., us-east-1) and cloud provider (aws, gcp, or azure)
  4. Update your .env file with the configuration shown above
  5. The system will automatically create your index with the correct dimensions
  6. Start using Flexible GraphRAG with Pinecone!
üí° Note: You don't need to manually create an index. Flexible GraphRAG will automatically create it with the correct embedding dimensions based on your LLM provider (e.g., 1536 for OpenAI's text-embedding-3-small, 384 for Ollama's all-minilm).
